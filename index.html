What is ProdOps?

ProdOps is the concept of measuring everything through the application development lifecycle. ProdOps is the missing component from requirements definition to application monitoring to ensure the application features meet your targeted outcomes. ProdOps enhances a lot of the current methodologies you leverage today.

  • Agile
  • Scrum
  • Lean
  • Kanban
  • 12 Factor
  • Microservices
  • Continuous Integration
  • Continuous Delivery
  • DevOps
  • ChatOps

The methodologies above describe the workflow for how source code could move throughout a company and eventually make it to a user on their computer, tablet, or phone.

The Workflow (Notes)

Agile

  1. Proposals are created
    • Proposals are rated
    • Dependencies assigned
    • Outcome associated
  2. Proposals are scheduled
    • Proposals are rated
    • Dependencies assigned
    • Outcome associated
  3. Developer self-assigns proposal
    • Proposals are rated
    • Dependencies assigned
    • Outcome associated
  4. Developer creates local branch
    • Proposals are rated
    • Dependencies assigned
    • Outcome associated
  5. Developer begins coding solution
    • Proposals are rated
    • Dependencies assigned
    • Outcome associated
  6. Developer commits incremental changes locally
    • Proposals are rated
    • Dependencies assigned
    • Outcome associated
  7. Developer pushes changes to server
    • Proposals are rated
    • Dependencies assigned
    • Outcome associated

DevOps

  1. Developer creates a pull request
    • Proposals are rated
    • Dependencies assigned
    • Outcome associated
  2. Continuous integration starts a build of the solution
    • Proposals are rated
    • Dependencies assigned
    • Outcome associated
  3. Status of the build is reported back to show status of solution
    • Proposals are rated
    • Dependencies assigned
    • Outcome associated
  4. Other engineers collaborate on proposed solution
    • Quality
    • Performance
    • Hardware Requirements
  5. Additional changes take place on the developers workstation
    • Proposals are rated
    • Dependencies assigned
    • Outcome associated
  6. Developer pushes changes to server
    • Proposals are rated
    • Dependencies assigned
    • Outcome associated
  7. Continuous integration starts a clean, new build
    • Proposals are rated
    • Dependencies assigned
    • Outcome associated
  8. Engineers discuss mergeability of solution into production branch
    • Proposals are rated
    • Dependencies assigned
    • Outcome associated
  9. Solution is automatically published to a production like environment for acceptance
    • Proposals are rated
    • Dependencies assigned
    • Outcome associated
